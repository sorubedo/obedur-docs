## 常见问题列表

<details>
<summary><b>变基后网络连接失败或无法识别？</b></summary>

**问题描述**：在从其他系统（如 Fedora Workstation）变基到 `obedur-os` 后，原有的 Wi-Fi 连接可能无法正常工作。

**原因分析**：为了提升无线网络的稳定性和效率，`obedur-os` 将 **NetworkManager** 的后端由默认后端的 `wpa_supplicant` 切换为了 **iwd (iNet wireless daemon)**。

**解决方法**：
1.  **忘记网络**：您需要先在系统中“忘记”当前的网络连接。
2.  **重新连接**：再次搜索并输入密码重新连接即可。

**推荐工具**：
- **图形化操作**：如果您已经进入 Niri 环境，可以使用 **noctalia-shell** 的网络组件进行操作。
- **命令行/终端**：如果 Niri 尚未配置完成，或者您在 TTY 下，请使用内置的交互式工具：
  ```bash
  wifitui
  ```

</details>

<details>
<summary><b>无法弹出密码请求对话框 (Polkit)？</b></summary>

**问题描述**：在运行需要管理员权限的程序（如图形化包管理器、系统设置修改等）时，系统没有弹出输入密码的授权对话框。

**原因分析**：为了保持系统的极简性，`obedur-os` 镜像本身并未内置通用的 Polkit 用户代理 (polkit agent)。

**解决方法**：
强烈建议您通过 **noctalia-shell** 的插件管理功能，安装 Noctalia 官方提供的 **"polkit agent"** 插件。该插件能够完美集成在您的桌面环境中，为您提供现代化的授权弹窗支持。

</details>

<details>
<summary><b>主目录中缺少“下载”、“图片”等标准文件夹？</b></summary>

**问题描述**：在安装或初始化系统后，发现用户主目录下没有 `Downloads`、`Pictures`、`Documents` 等标准的 XDG 用户目录。

**解决方法**：
您可以打开终端，执行以下命令来自动生成这些标准目录：
```bash
xdg-user-dirs-update
```
执行后，系统会根据当前的语言设置自动在主目录下创建相应的文件夹。

</details>
