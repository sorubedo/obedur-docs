在成功变基到 `obedur-os` 并重启系统后，您需要执行一些初始配置来确保系统的最佳体验。由于 `obedur-os` 是基于容器构建的，部分硬件相关的底层配置需要手动完成。

<Aside type="tip" title="首次配置推荐方式">
在 **tuigreet** (登录界面) 的会话选择菜单中，您可以先选择 **bash** 登录。

**原因**：由于 `obedur-os` 默认不为变基用户提供 Niri 初始配置，如果您直接登录 Niri，可能会发现快捷键无法打开终端。在 bash 环境下完成下方的同步和配置步骤后，再通过 `exit` 退出并重新登录 Niri 即可。
</Aside>

<summary><b>1. TTY字体设置</b></summary>

### 为什么需要手动设置？

**BlueBuild 镜像本身并不包含 TTY 字体的配置**。在传统的 Fedora 安装中，这些设置通常由安装程序一次性生成。

由于 Fedora 默认的 TTY 字体在许多高分辨率显示器上显得非常小，这会导致 **tuigreet** (登录管理器) 难以辨认。

### 解决方法

`obedur-os` 已经预装了 `terminus-fonts-console` 字体包，您可以执行以下操作来优化显示效果：

#### A. 临时切换 (立即生效)
在 TTY 终端执行：
```bash
setfont ter-v32b
```

#### B. 永久配置 (重启生效)
1. **编辑配置文件**：
   ```bash
   sudo nano /etc/vconsole.conf
   ```
2. **设置 FONT 变量**（将内容修改或添加为）：
   ```ini
   FONT="ter-v32b"
   ```
3. **保存并退出** (`Ctrl+O`, `Enter`, `Ctrl+X`)。

<Aside type="tip">
**推荐字体尺寸**：
- `ter-v16n`: 标准尺寸
- `ter-v24b`: 中等大粗体
- `ter-v32b`: 极大粗体 (推荐用于 4K 或高分屏)
</Aside>

</details>

<summary><b>2. 代理服务配置 (Sing-box)</b></summary>

<Aside type="caution">
**配置前提**：在开始之前，请确保您的网络连接正常。如果您在变基后发现 Wi-Fi 无法连接，请参考 [常见问题：变基后网络连接失败？](/obedur-docs/faq/#网络连接)。
</Aside>

### 预置说明

`obedur-os` 预置了来自 **ref1nd** 分支的 `sing-box`，该分支提供了更丰富的功能支持。系统中已经内置了一个默认的 TUN 模式配置文件。

### 配置步骤

1.  **设置订阅链接**：
    编辑默认的 TUN 配置文件：
    ```bash
    sudo nano /etc/sing-box/tun.json
    ```
    找到文件末尾附近的 `providers` 部分，将 `url` 中的 `<CHANGE ME>` 替换为您的实际订阅链接：
    ```json
    "providers": [
      {
        "type": "remote",
        "tag": "proxy",
        "url": "https://your-subscription-link-here",
        ...
      }
    ]
    ```

2.  **管理服务**：
    使用 `systemctl` 来启动或管理代理服务。服务名称为 `sing-box@tun.service`。
    - **启动并设置开机自启**：
      ```bash
      sudo systemctl enable --now sing-box@tun.service
      ```
    - **查看运行状态**：
      ```bash
      systemctl status sing-box@tun.service
      ```
    - **重启服务 (修改配置后)**：
      ```bash
      sudo systemctl restart sing-box@tun.service
      ```

3.  **访问控制面板**：
    系统预配置了 Web 面板，您可以通过浏览器访问：
    - **地址**：`http://127.0.0.1:23333`
    - **密码**：`obedur`

<Aside type="caution">
**注意**：首次启动时，sing-box 会自动下载面板文件和规则集，请确保网络连接正常（可能需要临时开启系统代理或等待下载完成）。
</Aside>

<summary><b>3. Flatpak 软件管理</b></summary>

### 预置说明

在 **BlueBuild** 架构中，Flatpak 软件并不会被包含在系统镜像中，而是通过一个**用户级服务**在系统启动时进行初始化和自动下载。

`obedur-os` 预置了以下两个核心管理工具：
- **Warehouse**：强大的 Flatpak 图形化管理工具，支持安装、卸载、清理以及管理远程仓库。
- **Flatseal**：细粒度的权限控制工具，用于管理 Flatpak 应用的系统资源访问权限。

### 管理工具：bluebuild-flatpak-manager

系统提供了一个便捷的命令行工具 `bluebuild-flatpak-manager` 来管理此服务。

- **查看当前配置**：
  ```bash
  bluebuild-flatpak-manager show
  ```
- **手动启动安装/应用配置**：
  ```bash
  bluebuild-flatpak-manager apply all
  ```
- **禁用自动安装服务** (如果您不打算使用 Flatpak 或网络环境极差)：
  ```bash
  bluebuild-flatpak-manager disable all
  ```
- **重新启用自动安装**：
  ```bash
  bluebuild-flatpak-manager enable all
  ```

### 重要提示

1.  **浏览器安装**：`obedur-os` 默认没有内置浏览器。建议在 `Warehouse` 安装完成后，通过它来安装您偏好的浏览器 (如 Firefox, Chrome 或 Brave)。
2.  **网络与代理**：由于 Flatpak 包是在首次进入系统后异步下载的，如果您在第一次启动时没有配置好代理，初始化可能会失败或非常缓慢。建议先完成 **Sing-box** 配置并确保网络畅通，再手动执行 `bluebuild-flatpak-manager apply all`。
3.  **可选性**：如果您不喜欢 Flatpak，可以随时通过上述命令将其彻底禁用。

<summary><b>4. Niri 配置初始化</b></summary>

### 预置说明

`obedur-os` 在 `/etc/skel/.config/niri` 中预置了一套深度的模块化配置，集成了 **Noctalia Shell** 和常用工具。
- **对于新创建的用户**：这些配置会在创建用户时自动应用。
- **对于从其他系统变基来的用户**：您需要手动同步这些预置配置到您的个人目录。

### 初始化与同步步骤

执行以下命令将完整的预置配置结构（包含 Niri、Ghostty 终端、Starship 提示符以及相应的 Bash 环境设置）应用到您的主目录：

```bash
# 创建必要的配置目录
mkdir -p ~/.config/niri ~/.config/ghostty

# 同步所有预置配置
cp -r /etc/skel/.config/niri/* ~/.config/niri/
cp /etc/skel/.config/ghostty/config ~/.config/ghostty/config
cp /etc/skel/.config/starship.toml ~/.config/starship.toml

# 同步 Bash 环境配置 (启用 Starship 的前提)
cp /etc/skel/.bashrc ~/.bashrc
cp /etc/skel/.bash_profile ~/.bash_profile
```

<Aside type="danger" title="重要警告">
上述命令会**直接覆盖**以下现有的配置文件：
- `~/.config/niri/` 完整配置结构
- `~/.config/ghostty/config` (Ghostty 终端配置)
- `~/.config/starship.toml` (Starship 提示符美化配置)
- `~/.bashrc` 和 `~/.bash_profile` (Bash 环境配置)

如果您之前在 `.bashrc` 中有自己的别名或设置，**请在执行前务必进行备份！**
</Aside>

<Aside type="caution" title="个性化修改建议">
**请注意**：这套预置配置是开发者根据个人习惯编写的，**并不一定完全适合您**。同步完成后，请务必根据您的硬件和喜好进行修改：

1.  **显示器配置 (`config.d/outputs.kdl`)**：
    - **必做**：开发者默认禁用了自定义显示器配置。
    - **操作**：在 Niri 会话中运行 `niri msg outputs` 查看可用显示器名称。编辑 `config.d/outputs.kdl` 填入您的配置，然后在 `config.kdl` 中取消 `include "config.d/outputs.kdl"` 的注释。
2.  **环境变量 (`config.d/env.kdl`)**：
    - **语言**：修改 `LANG` 等变量以符合您的语言偏好。
</Aside>

### 更新与重载配置

在修改配置文件后，您不需要重启系统：
- **实时重载**：保存文件后，Niri 通常会自动重载配置。
- **手动触发重载**：如果需要手动触发，可以执行：
  ```bash
  niri msg action do-reload-config
  ```

### 推荐安装的 Noctalia 插件

为了获得完整的桌面体验，强烈建议您在进入 Niri 后，通过 **Noctalia Shell** 的插件管理功能安装以下插件：

- **[Polkit Agent](https://noctalia.dev/plugins/polkit-agent/)**：**必装**。官方插件,用于处理系统权限请求弹窗。如果没有它，您将无法在图形界面下输入管理员密码（如在 Warehouse 中安装软件）。
- **[Notes Scratchpad](https://noctalia.dev/plugins/notes-scratchpad/)**：可选。开启强大的便签/随手记功能，通过 `Mod + N` 快速唤出。

### 常用快捷键与操作

`obedur-os` 预置了大量的快捷键以集成 **Noctalia Shell**。例如：
- `Mod + A` 或 `Space`：应用启动器
- `Mod + T`：打开终端 (**Ghostty**)
- `Mod + N`：切换便签抽屉 (需安装上述插件)
- `Mod + Q`：关闭窗口

<Aside type="tip">
**完整快捷键指南**：由于 Niri 的窗口和工作区管理功能非常强大且丰富，完整的快捷键列表及其详细说明请参考：[Niri 窗口管理器 - 预置快捷键指南](/obedur-docs/features/niri/#预置快捷键指南)。
</Aside>

<summary><b>5. 主题与外观配置</b></summary>

### 预置说明

`obedur-os` 提供了高度自定义的外观支持，您可以通过以下工具分别配置 GTK 和 Qt 应用的主题。

### 配置步骤

1.  **GTK 主题配置**：
    - **获取资源**：从 [gnome-look.org](https://www.gnome-look.org/) 下载您喜爱的 GTK 主题。
    - **安装路径**：解压并将其放入您的主目录下的 **`~/.themes`** 文件夹中（如果不存在请手动创建）。
    - **应用主题**：打开预装的 **nwg-look** 程序进行主题、图标及光标的设置。
    - **环境变量覆写**：由于 **Nautilus** 等部分应用可能不遵循常规配置，预置配置在 `~/.config/niri/config.d/env.kdl` 中提供了 GTK 主题的环境变量覆写示例（默认已注释）。设置好主题后，请前往该文件**取消相关注释**。

2.  **Qt 主题配置 (Qt5/6)**：
    - **获取资源**：从 [store.kde.org (Kvantum 分类)](https://store.kde.org/browse?cat=123) 下载 Kvantum 主题。
    - **安装路径**：通过 Kvantum Manager 的界面安装，或手动放入 `~/.config/Kvantum`。
    - **设置工具**：打开 **Kvantum Manager** (程序名：`kvantummanager`) 选择并应用主题。
    - **启用设置**：设置完成后，请确保在 `~/.config/niri/config.d/env.kdl` 中找到对应的 **Kvantum 环境变量** 并**取消注释**。

3.  **图标包配置**：
    - **获取资源**：从 [gnome-look.org (Icons 分类)](https://www.gnome-look.org/browse?cat=132) 下载图标包。
    - **安装路径**：解压并将其放入您的主目录下的 **`~/.icons`** 文件夹中（如果不存在请手动创建）。
    - **应用图标**：
        - **GTK 应用**：使用 **nwg-look** 进行设置。
        - **Qt5/6 应用**：分别打开 **qt5ct** 和 **qt6ct** 工具，在“图标主题”选项卡中选择对应的图标包。
        - **Noctalia 组件**：在 `~/.config/niri/config.d/env.kdl` 中，找到并取消关于图标主题的环境变量注释（例如 `// QS_ICON_THEME "Tela-dark"`），将其中的名称修改为您实际安装的**图标包名称**。

4.  **字体配置**：
    - **GTK 字体**：使用 **nwg-look** 程序进行设置。
    - **Qt5/6 字体**：分别打开 **qt5ct** 和 **qt6ct** 工具，在“字体”选项卡中进行设置。
    - **Noctalia 字体**：通过 **Noctalia Shell** 自带的设置界面进行字体个性化调整。

<summary><b>6. Flatpak 主题与权限配置</b></summary>

### 核心原理

由于 Flatpak 应用运行在沙盒中，您需要手动授予其访问主题文件的权限，并配置环境变量来强制应用主题。

### 配置步骤

1.  **授予全局权限**：
    执行以下命令，允许所有 Flatpak 应用读取您的本地主题、图标以及 Kvantum 配置：
    ```bash
    flatpak override --user --filesystem=~/.themes:ro --filesystem=~/.icons:ro --filesystem=xdg-config/gtk-3.0:ro --filesystem=xdg-config/Kvantum:ro
    ```

2.  **设置主题环境变量**：
    通过环境变量告知 Flatpak 应用具体使用哪个主题，并强制 Qt 应用使用 Kvantum 引擎：
    ```bash
    # 设置 GTK 主题名 (请替换为您实际的主题名称)
    flatpak override --user --env=GTK_THEME=您的主题名称

    # 强制 Qt 应用使用 Kvantum 引擎
    flatpak override --user --env=QT_STYLE_OVERRIDE=kvantum
    ```

3.  **安装 Flatpak 主题引擎**：
    为了让 Flatpak 应用能够识别并调用主题引擎，建议通过 **Warehouse** 或命令行安装以下运行时：
    - **Qt 支持**：`org.kde.KStyle.Kvantum`
    - **GTK 支持**：`org.gtk.Gtk3theme.adw-gtk3` (及对应的 dark 版本)

<Aside type="tip">
**进阶管理**：您也可以打开系统预装的 **Flatseal**，在 **"All Applications"** 的 **Filesystem** 和 **Environment** 栏目下直观地查阅和修改上述设置。
</Aside>
